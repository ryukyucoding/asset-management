防線一 — 靜態 + 執行期型別安全

TypeScript Strict Mode + Zod

✓  "strict": true
  • strictNullChecks — 消滅 null/undefined 執行期崩潰
  • noImplicitAny — 禁止隱式 any，強迫明確型別宣告
  • strictFunctionTypes — 函式參數逆變型別嚴格檢查
  • vue-tsc --build — 對 Vue SFC template 做完整型別驗證
  • Zod Runtime Validation — TS 只保護編譯期；Zod 在 API 邊界對所有外部輸入做 schema 驗證，確保執行期型別安全
// backend/tsconfig.json { "compilerOptions": { "strict": true, // ← 一鍵開啟所有嚴格規則 "target": "ES2022", "module": "commonjs", "paths": { "@domain/*" : ["./src/domain/*"], "@infrastructure/*" : ["./src/infrastructure/*"], "@services/*" : ["./src/services/*"] } } }
防線一 — 靜態分析（續）

雙層 Linter 架構

ESLint v10
  • eslint-plugin-vue — Vue 元件最佳實踐
  • @vue/eslint-config-typescript — Vue + TS 整合
  • eslint-plugin-vitest — 測試程式碼品質
  • eslint-plugin-playwright — E2E 測試規範
  • eslint-config-prettier — 與 Prettier 避免衝突
Oxlint Rust 50–100× 更快
  • 以 Rust 編寫，速度比 ESLint 快 50–100 倍
  • "correctness": "error" — 正確性問題即 CI 失敗
  • 涵蓋 6 個 plugins：eslint, typescript, unicorn, oxc, vue, vitest
// .oxlintrc.json { "plugins": ["eslint", "typescript", "unicorn", "vue"], "categories": { "correctness": "error" } }
防線二 — 格式統一

Prettier 格式規範

.prettierrc.json
{ "semi": false, "singleQuote": true, "printWidth": 100 }
  • 消除 PR diff 中純格式差異，讓 Code Review 聚焦邏輯而非排版
  • 前後端統一採用相同風格，零認知負擔切換工作區
  • 搭配 eslint-config-prettier 確保 Linter 不干擾格式化
前端 npm scripts
$ pnpm lint:oxlint # Oxlint fix-in-place $ pnpm lint:eslint # ESLint fix-in-place $ pnpm format # Prettier format

架構層面

Clean Architecture 防止 Code Smells

Middleware
@middleware/*
auth · rate-limit · idempotency
Services（Application）
@services/*
業務流程編排
Infrastructure
@infrastructure/*
DB · Redis · Queue
Domain（Core）
@domain/*
純商業邏輯，零框架依賴

透過 路徑別名 強制分層依賴方向，從根本防止：

  • God Class — 每層職責明確，不允許越層存取
  • 循環依賴 — 依賴方向單向（外層 → 內層）
  • 緊耦合 — Domain 只定義 Interface，Infrastructure 實作
  • 難以測試 — 分層讓 Unit Test 可輕鬆注入 Mock 依賴
